コード・コマンド早見表

学習資料に登場するコードやコマンドは、大きく4種類に分かれます。それぞれの実行場所と目的を押さえておきましょう。

1. Azure CLI コマンド

実行場所：ターミナル（コマンドプロンプト / PowerShell / Bash）
目的：Azureのリソース（Webアプリ、コンテナ、Key Vaultなど）をコマンドで作成・管理する

Azure CLI
az webapp create \
  --resource-group myRG \
  --plan myPlan \
  --name myApp \
  --runtime "DOTNETCORE:8.0"
ポイント Azureポータル（Web画面）でマウス操作する代わりに、同じ操作をコマンドで実行できるツールです。使用前に az login でサインインが必要です。

2. C# コード（.NET SDK）

実行場所：Visual Studio / VS Code で作成するC#アプリケーション内
目的：自分が作るアプリのコードからAzureサービス（Blob Storage、Cosmos DB、Key Vault等）を操作する

C#
BlobServiceClient serviceClient = new BlobServiceClient(connectionString);
BlobContainerClient containerClient = serviceClient.GetBlobContainerClient("my-container");
await blobClient.UploadAsync("file.txt", overwrite: true);
ポイント dotnet new console でプロジェクトを作成し、NuGetパッケージ（例：Azure.Storage.Blobs）をインストールしてから使用します。

3. CRON式

実行場所：C#コード内の [TimerTrigger(...)] 属性の引数（Azure Functionsの一部）
目的：Azure Functionsを「いつ自動実行するか」スケジュール指定する
形式：{秒} {分} {時} {日} {月} {曜日}（6フィールド）

C# - Azure Functions
// 5分ごとに実行
[TimerTrigger("0 */5 * * * *")]

// 毎日午前9時に実行
[TimerTrigger("0 0 9 * * *")]

// 平日の毎朝8時30分に実行
[TimerTrigger("0 30 8 * * 1-5")]
ポイント CRON式は独立したコマンドではなく、C#コードに組み込むスケジュール設定値です。Linuxのcronと似ていますが、先頭に「秒」が追加された6フィールド形式です。

4. KQL（Kusto Query Language）

実行場所：Azureポータル → Application Insights → ログ（Log Analytics）画面
目的：アプリの監視データ（リクエスト、例外、パフォーマンス）を検索・分析する

KQL
requests
| where timestamp > ago(24h)
| where success == false
| summarize count() by resultCode
| order by count_ desc
ポイント KQLはAzureポータルのブラウザ上で直接入力・実行します。SQLに似た文法ですが、パイプ（|）でデータを絞り込んでいく形式が特徴です。

まとめ

種類 実行場所 目的
Azure CLI az ... ターミナル Azureリソースの作成・管理
C# / .NET SDK アプリのソースコード内 アプリからAzureサービスを操作
CRON式 C#コードの [TimerTrigger] 内 Azure Functionsの定期実行スケジュール
KQL Azureポータルのログ画面 監視データの検索・分析
試験でのポイント AZ-204では、これらを実際に実行するのではなく「このシナリオではどのコード/コマンドが正しいか？」を選択肢から選ぶ形式で問われます。各コードの意味と使い分けを理解しておくことが重要です。
